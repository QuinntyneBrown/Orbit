# L1 - High-Level Requirements

> **Scope:** This document defines the high-level requirements for **Orbit**, a career management and job search automation platform. Requirements cover resume content management, document generation, application pipeline tracking, job discovery, offer evaluation, outreach, and workflow automation.

---

## L1-001: Resume Content Management

The system must maintain a single source of truth for all resume content in version-controlled Markdown files, support base and tailored variants, and enforce consistent structure across all content artifacts.

---

## L1-002: Document Generation

The system must convert Markdown resume source files into professional-quality Word (.docx) and PDF outputs suitable for submission to recruiters and applicant tracking systems (ATS), without requiring Microsoft Word for PDF generation.

---

## L1-003: Application Pipeline Tracking

The system must track every role applied to through a defined lifecycle of states, linking each entry to the originating tailored resume, the job source, application date, compensation details, and current status.

---

## L1-004: Offer Evaluation

The system must support structured, scored evaluation of job postings before any tailoring effort is invested, using a defined set of weighted dimensions, and must store evaluation artifacts per role in a consistent format.

---

## L1-005: Job Search and Discovery

The system must search relevant job boards, company career portals, and recruiter-specific boards for open roles matching the configured candidate profile, and must surface results in a structured, actionable format.

---

## L1-006: Search History and Deduplication

The system must persist job search results across runs, deduplicate listings by company and title, and surface net-new postings and status changes relative to previous runs.

---

## L1-007: Compensation Research

The system must research and report market rate estimates for role postings that do not list compensation, drawing on publicly available salary data sources and comparable listings.

---

## L1-008: Role Archetype Classification

The system must classify every discovered job posting into a defined set of role archetypes that reflect the candidate's positioning strategy, and must use archetype to inform evaluation weighting and tailoring emphasis.

---

## L1-009: Outreach Management

The system must generate and organize LinkedIn messages, email drafts, and recruiter follow-ups in a dedicated location, drawing on existing message examples as style references and on offer evaluation context for personalization.

---

## L1-010: Interview Preparation

The system must maintain a cumulative bank of STAR+Reflection behavioral stories linked to the candidate's work history, surfacing relevant stories during offer evaluation and after reaching the interview stage.

---

## L1-011: Recruiter Relationship Management

The system must track recruiter and staffing vendor relationships including contact information, last contacted date, current engagement status, and priority tier, integrated with the application pipeline.

---

## L1-012: Workflow Automation

The system must automate multi-step workflows including variant creation, batch tailoring, document build, and base resume sync verification, reducing manual steps and supporting parallel execution during high-volume search periods.

---

## L1-013: Session Integrity

The job search module must validate its runtime environment at startup — confirming the base resume exists, is current, and the profile data read is live rather than stale — before executing any search or evaluation.

---

## L1-014: Search Result Persistence and Export

The job search module must write each search run's results to a dated file, maintain a rolling history window, and produce a diff summary comparing the current run to the previous one.

---

## L1-015: Data Integrity and Privacy

All data files containing personal information, compensation targets, contact details, and application outcomes must be protected from accidental exposure (e.g., excluded from public repositories), and all system operations must handle candidate data without transmitting it to unintended third parties.
